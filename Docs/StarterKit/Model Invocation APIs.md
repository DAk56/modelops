
ModelOps provides two primary invocation routes to accommodate varying levels of user control and configuration:

- **Convenience Routes**: Simplified usage with sensible defaults for quick integration and quick usage.
- **Proxy Routes**: Full control over model parameters for advanced use cases.

## Model Reference Table

ModelOps supports multiple AI models for tasks like text generation and document analysis. Depending on the API route (Convenience or Proxy), you may need to use either a `model_identifier` or a `model_id`.

   |Service	|Model Name {{model_identifier}}|	Model ID {{model_id}}|
   |:-|:-|:-|
   |Bedrock	|amazon-titan-text-lite|	amazon.titan-text-lite-v1|
   |Bedrock|	anthropic-claude-3-haiku|	anthropic.claude-3-haiku-20240307-v1:0|
   |Bedrock|	anthropic-claude-3-sonnet|	anthropic.claude-3-sonnet-20240229-v1:0|
   |Bedrock|	meta-llama3-8b-instruct	|meta.llama3-8b-instruct-v1:0|
   |Comprehend| comprehend| comprehend|
   |Textract|	textract|	textract|

- Refer to the specific API documentation to determine which identifier to use.

!!!Note
    Models are accessible only to users who are entitled to them. Ensure entitlement is granted before attempting invocation or metadata retrieval.

> ### Invoke Bedrock (Convenience Route) API

**Purpose**:

Ideal for basic text generation tasks. Requires only a prompt; default model configurations can be overridden if needed.

**Endpoint**:

`/v1/models/bedrock/{{model_identifier}}`

**Request**:

   Send a POST request to the endpoint given.

   - Header:
     - `Authorization`: Bearer token or Basic Auth credentials
     - `tenant_Id`: Identifier for the project or client (User do not need this header if the token is generated via client credentials API)
   - Path parameter:
     - `model_identifier`: The model identifier for which the status is needed.

The body of the request can be of following format. 

```json
    {
      "prompt": "extract the data from this document and give me a summary" 
    }
```
**Response**:
``` json
   {
       "jobId": "amazon.titan-text-lite-v1_xxxxxxx",
       "modelId": "amazon.titan-text-lite-v1",
       "jobStatus": "Queued",
       "createdAt": "2025-08-18T12:54:24.878Z"
   }
```
> ### Invoke Textract (Convenience Route) API

**Purpose**:

With a set of queries users can extract the data given in a document. There are two path to extract the data.

#### Path 1: Standard Flow (for Files size ≤ 6 MB)

**Endpoint**:

`/v1/models/textract`

**Request**:

   Send a POST request to the endpoint given. 

   - Header:
     - `Authorization`: Bearer token or Basic Auth credentials
     - `tenant_Id`: Identifier for the project or client (User do not need this header if the token is generated via client credentials API)

   The body of the request can be of following format.
```json   
   {
       "type":[
               "forms", "queries","tables", "layout", "signatures"
           ],
       "queries": [
                           {"Alias": "full_name", "Pages": ["1"], "Text": "What is the full name on the license?"},
                           {"Alias": "address", "Pages": ["1"], "Text": "What is the address on the license?"},
                           ...
                           ],
       "attachment":{
           "type": "png",
           "base64EncodedData":" <Base64 encoded image data>"
   
       }
   }
```
**Response**:
```json
   {
       "jobId": "textract_xxxxxxx",
       "modelId": "textract",
       "jobStatus": "Queued",
       "createdAt": "2025-08-18T13:39:09.157Z"
   }
```

#### Path 2: Large File Flow (for Files size > 6 MB)
***Step 1: Prepare API***

**Endpoint**:

`/v1/prepare/{{model_id}}`

**Request**:

   Send a POST request to the endpoint given. 

   - Header:
     - `Authorization`: Bearer token or Basic Auth credentials
     - `tenant_Id`: Identifier for the project or client (User do not need this header if the token is generated via client credentials API)
   - Path parameter:
     - `modelId`: The model id for which the file need to be prepared(textract).

   The body of the request can be of following format.

```json
    {
        "documentType":"pdf"
    }
```

**Response**:

```json

{
  "jobId": "textract_xxxxxxx",
  "modelId": "textract",
  "url": "https://presigned-upload-url"
}
```

***Step 2: Upload File***

**Endpoint**:

The URL generated by the prepare API is the endpoint.

**Request**:

   Send a PUT request with the file in **binary format**.

**Response**:

HTTP 200 OK

***Step 3: Invoke Textract***

**Endpoint**:

`/v1/models/textract/{{jobId}}`

**Request**:

   Send a POST request to the endpoint given. 

   - Header:
     - `Authorization`: Bearer token or Basic Auth credentials
     - `tenant_Id`: Identifier for the project or client (User do not need this header if the token is generated via client credentials API)

   - Path parameter:
     - `jobId`: The job id generated by the Prepare API.

   The body of the request can be of following format.

```json
{
    "methodName": "analyze_id"
}
```

**Response**:

```json
{
    "jobId": "textract_xxxxx",
    "modelId": "textract",
    "jobStatus": "Queued",
    "createdAt": "2025-08-18T12:54:24.878Z"
}
```
> ### Invoke Bedrock (Proxy Route) API

   **Purpose**:

   Provides granular control over model behavior, including inference configuration and multimodal input.

**Endpoint**:

`/v1/proxy/aws/bedrock/{{model_id}}`

**Request**:

   Send a POST request to the endpoint given. 

   - Header:
     - `Authorization`: Bearer token or Basic Auth credentials
     - `tenant_Id`: Identifier for the project or client (User do not need this header if the token is generated via client credentials API)
   - Path parameter:
     - `modelId`: The model id for which the status is needed.

   The body of the request can be of following format.

```json
   {
       "methodName": "converse",
       "parameters": {
           "body": {
               "inferenceConfig": {
                   "maxTokens": 512,
                   "temperature": 0.5,
                   "topP": 0.9
               },
               "messages": [
                   {
                       "role": "user",
                       "content": [
                           {
                               "text": "What is this document"
                           },
                           {
                               "image": {
                                   "format": "a",
                                   "source": {
                                      "bytes": "Base64 encoded data"
                                   }
                               }
                           }
                       ]
                   }
	      	]
           }
       }
   }
```
**Response**:
```json
   {
       "jobId": "amazon.titan-text-lite-v1_xxxxxxx",
       "modelId": "amazon.titan-text-lite-v1",
       "jobStatus": "Queued",
       "createdAt": "2025-08-18T12:54:24.878Z"
   }
```

   Allows users to configure Textract-specific features and queries for document analysis.

`/v1/proxy/aws/textract`

**Request**:

   - Header:
     - `Authorization`: Bearer token or Basic Auth credentials

   The body of the request can be of following format.
```JSON
   {
       "methodName": "startDocumentAnalysis",
       "parameters": 
       {
           "featureTypes": [
               "forms","tables","signatures","layout","queries"
           ],
           "document": 
           {
               "type": "png",
               "base64EncodedData": "Base64 Encoded data"
           },
           "queriesConfig": {
               "Queries": [
                   {
                       "Alias": "full_name",
                       "Pages": [
                           "1"
                       ],
                       "Text": "What is the full name on the license?"
                   },
                   {
                       "Alias": "address",
                       "Pages": [
                           "1"
                       ],
                       "Text": "What is the address on the license?"
                   },
                   ...
               ]
           }
       }
    }
```
**Response**:
```JSON
   {
       "jobId": "textract_xxxxxxx",
       "modelId": "textract",
       "jobStatus": "Queued",
       "createdAt": "2025-08-18T13:50:50.078Z"
   }
```

> ### Invoke Comprehend (Proxy Route) API

   **Purpose**:

   Invokes Comprehend service for NLP tasks such as entity detection, sentiment analysis, or language detection.

**Endpoint**:

`/v1/proxy/aws/comprehend`

**Request**:

   Send a POST request to the endpoint given. 

   - Header:
     - `Authorization`: Bearer token or Basic Auth credentials
     - `tenant_Id`: Identifier for the project or client (User do not need this header if the token is generated via client credentials API)

   The body of the request can be of following format.

```json
{
        "methodName": "detect_sentiment",
        "parameters": {
        "body": {
                    "Text":"<Input Text>",
        "LanguageCode":"en"
    }
 }

}   
``` 

**Other Supported Methods**:

- detect_sentiment
- detect_entities
- detect_key_phrases
- detect_syntax
- detect_dominant_language
- detect_pii_entities
- detect_dominant_languages
- detect_toxic_content
- contains_pii_entities

!!! notes
    - The `methodName` determines which Comprehend feature is invoked.
    - Ensure `LanguageCode` is provided for text-based operations.

**Response**:
```json
{
    "jobId": "comprehend_xxxxxxxxx",
    "modelId": "comprehend",
    "jobStatus": "Queued",
    "createdAt": "2025-11-05T14:45:46.407Z"
}
```


!!! Notes
    - All invocation APIs return a consistent response structure containing:
        - jobId: Unique identifier for the submitted job
        - modelId: Identifier of the model used
        - jobStatus: Current status of the job (e.g., Queued, Completed)
        - createdAt: Timestamp of job creation
    - Ensure that:
        - The request format is valid.
        - The user has entitlement to the requested model.
        - Required headers and parameters are correctly specified.
