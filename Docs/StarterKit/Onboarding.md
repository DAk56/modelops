

To begin using ModelOps for your AI/ML workflows, each project must be onboarded into the system. This ensures proper access control, resource allocation, and usage tracking.

### Prerequisites
Before initiating onboarding for ModelOps, ensure the following:

1. **Tenant Information**: Prepare the tenant's email address and name.

2. **Model Requirements**: Identify which models will be used in the project.

!!!note
    To know more about available models, refer to the [Models Supported](About.md#multi-model-support) section.

### Required Information

The following details must be submitted to onboard a project:

- **Email Address**: Must be associated with Maximus account.
- **Tenant Name**: The logical workspace under which the project will be registered
- **List of AI Models**: Specify the models your project will use (e.g., text generation, classification, summarization)

### Access Provisioning

Once the onboarding request is reviewed:

- The project will be linked to the specified tenant.
- Access to the requested models will be granted based on entitlement policies.
- API credentials (Client ID and Client Secret) will be issued.

### How to Submit an Onboarding Request

Submit your onboarding request to <ModelOps-Support@maximus.com> with the required details.

Once the request is submitted, the ModelOps team will create the Client id and secret along with all permissions.
